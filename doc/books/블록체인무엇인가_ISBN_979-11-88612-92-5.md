#### 4단계 : 블록체인에게 떨어진 미션
> - 담론 : 순수 분산 P2P 환경의 무결성 유지는 얼마나 어려운가?, 순수 분산 P2P 환경의 무결성을 유지하는 방법은 무엇인가?
> - 요약 : 블록체인은 순수 분산 아키텍쳐에서 임의의 전체 노드 수와 각 노드의 신뢰성을 확인하여 무결성을 확보해야 하는 문제를 포함한다., 순수 분산 아키텍쳐의 무결성을 확보하기 위해서는, 전체 노드의 개수와 각 노드의 신뢰성을 알 수 있어야 한다.
> - 키워드 : 신뢰, 무결성, 상호작용, 그룹역학, 합의 알고리즘, 기술적 결함, 악의적 노드
#### 5단계 : 그래서 블록체인이 뭔가요?
> - 담론 : 블록체인이란?
> - 요약 : 블록체인이란 무결성을 확보하고 유지하기 위해 순서에 따라 연결된 블록들의 정보 내용을 암호화 기법과 보안기술을 이용해 협상하는 알고리즘으로 구성된 소프트웨어 요소를 활용하는 원장의 순수 분산 P2P 시스템을 의미한다.(임시정의)
> - 키워드 : 데이터 구조, 알고리즘, 기술묶음, 일반 응용분야를 가지는 순수 분산 P2P 시스템 그룹
#### 6단계 : 소유권의 본질 이해하기
> - 담론 : 소유권의 증명과 블록체인간의 관계성은?
> - 요약 : 중앙 통제 아키텍쳐의 단일 원장은 신뢰성에 중대한 결함을 가진다., 다수가 동의하는 개별 원장의 그룹화를 통해 소유권의 신뢰성을 높일 수 있다., 블록체인 알고리즘은 개별 노드들이 하나의 일관된 소유권 상태에 도달하도록 한다., 식별, 인증, 승인은 암호화 기법을 통해 구현한다., 무결성은 소유권의 판단과 이전을 가능하게 한다.
> - 키워드 : 소유자 식별, 소유대상 식별, 소유자와 소유대상의 매핑, 증명서, 무결성, 신뢰성, 소유권
#### 7단계 : 이중사용, 블록체인이 해결한다
> - 담론 : 이중사용의 유형?
> - 요약 : 디지털 재화 복사 문제, 원장 분산 P2P 시스템에서의 문제, 분산 P2P 시스템의 무결성 침해 문제, 블록체인은 이중사용의 많은 부분을 해결하는 기술묶음임.
> - 키워드 : 이중사용
### 셋째마당 : 블록체인은 어떻게 작동하는가?
#### 8단계 : 블록체인의 청사진 그리기
> - 담론 : 순수 분산 P2P 시스템으로 소유권 관리를 위해 고려해야 할 사항은?
> - 요약 : 소유권 명세, 소유권 보호, 트랜잭션 데이터 저장, 신뢰할 수 없는 환경에 배분할 원장 준비, 원장 배분, 원장에 새 트랜잭션 추가, 어느 원장이 진실인지 판단
> - 키워드
#### 9단계 : 소유권 기록에서 모든것이 시작된다
> - 담론 : 소유권 명세 기록의 요소는?, 트랜잭션 무결성의 필요요소는?
> - 요약 : 소유권 이전 내역을 기술하고 소유권 이전 내역을 유지한다. 소유권 이전 내역을 계속 유지함으로 인해 특정 시점의 소유권 목록을 확인할 수 있다., 블록체인 트랜잭션에는 형식적 정확성, 의미상 정확성, 승인이 필요하다.
> - 키워드
#### 10단계 : 데이터 해싱하기
> - 담론 : 트랜잭션 기록을 위해 데이터를 암호화하는 방법은?, 데이터를 해싱하는 패턴은?
> - 요약 : 암호화 과정을 해싱이라 하며, 모든 데이터에 대한 해시값이 즉시 제공되어야 하고, 해시값은 확정적이어야 하고(1:1 대응), 의사 난수이어야 하며(정의역 예측 불가능), 비가역함수에 의한 일방적 사용이어야 하며(정의역 복원 불가능), 충돌 회피가 가능해야 한다(치역 중복 방지)., 반복해싱, 독립해싱, 결합해싱, 순차적해싱, 계층적해싱
> - 키워드 : 해싱, 데이터 해싱, 암호화, SHA256, 축약형
#### 11단계 : 해시값은 어디서 어떻게 사용될까?
> - 담론 : 기록된 해시값의 용도는?
> - 요약 : 데이터 비교, 바뀌면 안되는 데이터가 변경되었는지 감지, 변경-감지 방식으로 데이터를 참조, 변경-감지 방식으로 데이터를 저장, 계산량이 많이 필요한 과제 생성
> - 키워드 : 해싱, 자리수, 0, 난이도, 체인방식, 트리방식, 머클트리
#### 12단계 : 암호화 기법을 소개합니다
> - 담론 : 블록체인에서 암호화가 사용되는 부분은?
> - 요약 : 계정을 식별하기 위한 계정공개키, 트랜잭션 승인을 위한 암호문 생성시 계정공개키를 사용하며, 제3자는 계정공개키를 통해 트랜잭션 내용을 조회하여 증명할 수 있다.
> - 키워드 : 비대칭 암호화
#### 13단계 : 노드 여러분, 트랜잭션을 승인합니까?
> - 담론 : 디지털 서명의 역할은?, 트랜잭션 데이터 송신의 절차는?
> - 요약 : 유일한 서명자를 식별하고, 서명자가 문서 내용에 동의하고 그 내용의 실행을 승인했음을 명시하는 역할을 한다., 트랜잭션에 필요한 내용을 기술하고, 이를 해싱한 뒤, 해싱된 값을 개인키로 암호화하한 값을 기내용과 함께 첨부하여 비대칭 검증을 가능하게 한다.
> - 키워드 : 디지털 서명
#### 14단계 : 블록체인-데이터-구조를 만들어봅시다
> - 담론 : 블록체인-데이터-구조는 어떤 형태인가?, 블록체인-데이터-구조는 어떤 의미를 갖는가?
> - 요약 : 블록, 블록 헤더(블록간 참조 체인방식), 트랜잭션 데이터(머클트리방식), 가장 마지막 블록을 통해 역순으로 데이터를 탐색할 수 있으며, 데이터의 변경-감지 및 디지털 서명에 의한 접근제한이 가능하다.
> - 키워드 : 헤드, 헤더, 블록, 순서, 정렬, 해시참조
#### 15단계 : 잘 뜨개질된 블록체인, 어떻게 변경할까?
> - 담론 : 일부 트랜잭션 데이터를 변경할 수 있는 방법은?, 블록 데이터의 변경은 용이한가?
> - 요약 : 머클트리, 트리해시, 블록 해시, 해당 블록 해시를 참조하는 헤드 블록의 해시 등 연쇄되는 모든 해시를 모두 바꾸어야 한다., 모두 바꾸지 않으면 전체 시스템이 동작하지 않는다.(All or Nothing) 변용이 매우 어려운 해시구조의 속성에 기인한다.
> - 키워드
#### 16단계 : 블록체인이 데이터를 보호하는 방법
> - 담론 : 블록의 조작은 용이한가?, 블록헤더가 가지고 있어야 하는 정보는?
> - 요약 : 참조 해시 및 블록 헤더의 연쇄로 인해 여러 블록을 동시에 변조하기는 쉽지 않음., 직전 블록의 헤더를 가리키는 해시참조, 트랜잭션 데이터를 갖고 있는 머클트리의 루트, 해당 해시 퍼즐의 난이도, 해시 퍼즐 풀이를 시작한 시각, 해시 퍼즐을 해결할 난스
> - 키워드 : 조작, 해시, 난스, 헤더, 해시 퍼즐, 해시참조
#### 17단계 : 컴퓨터들이 정보를 배분하는 방법
> - 담론 : 피어간 정보교환의 속성은?, P2P 시스템의 노드 간 통신의 특성은?, P2P 시스템의 노드 간 통신의 목적은?
> - 요약 : 모든 피어는 인터넷을 통해 연결됨, 각 피어는 고유주소로 식별됨, 각 피어는 독립적으로 네트워크를 연결하고 단절할 수 있음, 각 피어는 독립적인 피어리스트를 가짐, 노드 간 통신은 메시지를 이용하여 이루어짐, 메시지는 고유한 인터넷 주소를 이용해 하나의 노드에서 다른 노드로 전송됨, 메시지가 목적지 주소에 도달한다는 보장이 없음(손실 가능성), 메시지는 중복되어 도달할 수 있음, 메시지는 전송한 순서와 다른 순서로 도착할 수 있음, 기존 연결 유지, 새로운 (노드)연결 수립, 새로운 (블록)정보 배포
> - 키워드 : 메시지, 소문, 피어, 뉴스, 손실가능성, 뉴스, 해시, 식별, 타임스탬프, 지속적인 방식 배포, 갱신 방식 배포, 온보딩 절차의 일부로 갱신 배포
#### 18단계 : 블록체인 속 무한경쟁 사회
> - 담론 : 새 블록에 대한 노드의 부정행위 유형은?, 노드의 부정을 막기 위한 블록체인-알고리즘의 원칙은?
> - 요약 : 노드 담합에 의한 트랜잭션 신뢰성 파괴(블록체인-데이터-구조로 해소됨), 다른 사람으로 위장(비대칭 암호화 및 디지털 서명을 통한 식별, 인증, 승인) 등, 검증규칙(트랜잭션 데이터, 블록 헤더 검증규칙), 보상, 처벌(즉시 보상 회수 또는 사후 보상 미지급), 경쟁(해시 퍼즐 해결에 대한 속도 경쟁, 품질 경쟁), 피어 통제(각 노드는 항상 둘 중 하나의 상태를 갖는다. 1)피어에 의해 생성되고 제출된 새 블럭을 평가 중 2)스스로 새 블록을 만들어 다른 피어에게 평가받으려 열심히 노력 중)
